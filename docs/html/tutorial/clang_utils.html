Clang Utils

Submodule clang_utils is a low-level API taking advantage of Clang and its python binding libclang, making it easier to extract program information from C/CPP source code.

Example files are stored in docs/source/examples.

Loading Source Code & Printing AST

Now, we have C file assets/load_ast.c

1int a;
2
3int main()
4{
5    int b, c;
6    return b + c;
7}

The Python script load_ast.py can parse this C file and dump AST.

Note that sys.path.append expression is used to let the interpreter know where MetronomeStatic package is placed when in development. If you have installed MetronomeStatic by pip, expression sys.path.append is not necessary. This is also true for the other Python scripts below.

 1import json
 2import sys
 3
 4sys.path.append("../../..")
 5
 6from MetronomeStatic import abspath_from_current_file
 7from MetronomeStatic.clang_utils import parse_file, extract_ast, get_func_decl
 8
 9# Load file by clang and get the translation unit
10translation_unit = parse_file(abspath_from_current_file("assets/load_ast.c", __file__))
11
12# Get the cursor from Translation Unit
13cursor = translation_unit.cursor
14
15# Get the function declaration of "main"
16main_func = get_func_decl(cursor, "main")
17
18# Extract the JSON representation of Clang AST
19ast = extract_ast(main_func)
20
21# Print AST
22print(json.dumps(ast, indent=2))

The output message below (some lines were omitted) showed that we could convert Clang AST of function main to JSON in this way.

{

    ...
    "spelling": "main",
    "children": [
        {
            ...

                        {
                            "kindText": "CursorKind.VAR_DECL",
                            ...
                            "spelling": "b"
                        },
                        {

Querying The AST Interactively

Querying AST can be annoying when handling a lot of recursions and loops. In this framework, inspired by Joern, a Scala-based code-analysis framework, we developed a functional API to access the AST.

For C file assets/interactive_query.c shown below:

1int f(int a, int b)
2{
3    return a * b;
4}
5
6int main(int paramA, int paramB)
7{
8    return paramA + paramB + f(paramA, paramB);
9}

To get the AST of function main, we could write code like this:

 1import sys
 2
 3sys.path.append("../../..")
 4
 5from MetronomeStatic.clang_utils import *
 6from MetronomeStatic import *
 7
 8path = abspath_from_current_file("assets/interactive_query.c", __file__)
 9
10"""
11Code above is critical for running this example,
12and not necessary when using in an interactive environment (IPython/Jupyter).
13
14↓↓ JUST FOCUS ON THE FOLLOWING CODE! ↓↓
15"""
16
17ret = (
18    iter_ast(parse_file(path).cursor)
19    .filter(lambda c: c.kind == CursorKind.FUNCTION_DECL and c.spelling == "main")
20    .head()
21)
22
23print(ret.kind, ret.spelling)

The output is:

1CursorKind.FUNCTION_DECL main

Queries for Variables

Now, we have C file assets/query_variables.c.

1int a;
2
3int demo_1(int d)
4{
5    int b, c;
6    return a + b + c + d;
7}

The Python script query_variables.py can parse this C file and query variables in this file or specific function.

In this script, we took some advantage of functional programming.

  • For functions like get_local_var_defs, it returns a generator SkyGenerator, yielding a clang.cindex.Cursor object for each iteration.

  • This generator supports some common patterns of functional programming, such as map, filter and head. Visit here for details.

  • Method .attribute("spelling") is a shortcut for .map(lambda node: node.spelling)

 1import sys
 2
 3sys.path.append("../../..")
 4from MetronomeStatic import abspath_from_current_file
 5from MetronomeStatic.clang_utils import (
 6    get_func_decl,
 7    get_local_var_defs,
 8    get_var_refs,
 9    get_param_decls,
10    get_global_ref_names,
11    parse_file,
12    get_func_decl,
13    all_globals,
14)
15
16
17# Load file by clang and get the translation unit
18translation_unit = parse_file(
19    abspath_from_current_file("assets/query_variables.c", __file__)
20)
21
22# Get the cursor to whole file from Translation Unit
23cursor = translation_unit.cursor
24
25# Get the cursor to function demo_1
26func_decl = get_func_decl(cursor, "demo_1")
27
28## VARIABLE DECLARATIONS
29
30# extract global variables defined in file
31global_variables = all_globals(cursor).attributes("spelling").to_set()
32print("Global variables:", global_variables)
33
34# extract local variables
35local_variables = get_local_var_defs(func_decl).attributes("spelling").to_set()
36print("Local variables:", local_variables)
37
38# extract parameters
39param_variables = get_param_decls(func_decl).attributes("spelling").to_set()
40print("Param variables:", param_variables)
41
42## VARIABLE REFERENCES
43
44# extract variable References
45referenced_variables = get_var_refs(func_decl).attributes("spelling").to_set()
46print("Referenced variables:", referenced_variables)
47
48
49# extract global references
50global_references = get_global_ref_names(func_decl).to_set()
51print("Used global variables:", global_references)

As the output message shown below:

  • In file assets/query_variables.c, global variable a was defined.

  • Function demo_1 defined local variables b and c inside, had parameter variable d, and referenced global variable a.

1Global variables: {'a'}
2Local variables: {'c', 'b'}
3Param variables: {'d'}
4Referenced variables: {'c', 'a', 'b', 'd'}
5Used global variables: {'a'}

Parsing Expressions

Now, we have C file assets/parse_expression.c. This file contains a series of operators.

 1int a;
 2
 3int demo(int d)
 4{
 5    int b, c;
 6    c = a + b;
 7    d = &a;
 8    d++;
 9    return c * d;
10}

The Python script parse_expression.py can parse all binary/unary operators inside function demo:

 1import sys
 2
 3sys.path.append("../../..")
 4from MetronomeStatic import abspath_from_current_file
 5from MetronomeStatic.clang_utils import (
 6    get_func_decl,
 7    parse_file,
 8    get_func_decl,
 9    CursorKind,
10    split_binary_operator,
11    split_unary_operator,
12    print_tokens,
13)
14
15
16# Load file by clang and get the translation unit
17translation_unit = parse_file(
18    abspath_from_current_file("assets/parse_expression.c", __file__)
19)
20
21# Get the cursor to whole file from Translation Unit
22cursor = translation_unit.cursor
23
24# Get the cursor to function demo
25func_decl = get_func_decl(cursor, "demo")
26
27# Walk the AST and parse the BINARY_OPERATOR/UNARY_OPERATOR
28for node in func_decl.walk_preorder():
29    if node.kind == CursorKind.BINARY_OPERATOR:
30        left, op, right = split_binary_operator(node)
31        print(
32            f"BINARY_OP left: {print_tokens(left)}  op: {op}  right: {print_tokens(right)}",
33        )
34    elif node.kind == CursorKind.UNARY_OPERATOR:
35        op, expr, pos = split_unary_operator(node)
36        print(f"UNARY_OP op: {op}  expr: {print_tokens(expr)}  position: {pos}")

As the output shown below:

  • For unary operator, the third returning parameter of split_unary_operator (local variable pos), indicated the position of operator.

    • UnaryOpPos is an Enum with optional values UnaryOpPos.AFTER and UnaryOpPos.BEFORE.

    • UnaryOpPos.AFTER indicates that the operator appears after the operated expression, such as p++

    • UnaryOpPos.BEFORE indicates that the operator appears before the operated expression, such as ++i, &p

  • Assignment operator is one of binary operator

1BINARY_OP left: c  op: =  right: a + b
2BINARY_OP left: a  op: +  right: b
3BINARY_OP left: d  op: =  right: & a
4UNARY_OP op: &  expr: a  position: UnaryOpPos.BEFORE
5UNARY_OP op: ++  expr: d  position: UnaryOpPos.AFTER
6BINARY_OP left: c  op: *  right: d
